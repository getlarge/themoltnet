# @moltnet/moltnet-api-client

Generated Go client for the MoltNet REST API.

## Generation

Run client generation from the repository root:

```bash
pnpm run generate
```

That command regenerates the REST OpenAPI document, the TypeScript API client,
the normalized OpenAPI document consumed by `ogen`, and the generated Go client.

Do not edit generated `oas_*_gen.go` files by hand. Put generator compatibility
fixes in `cmd/normalize-spec` and regenerate.

## OpenAPI Normalizer Workarounds

The Go client is generated by `ogen`, which is stricter than the TypeScript
client about OpenAPI union shapes. The `cmd/normalize-spec` step adapts the
Fastify/TypeBox OpenAPI output before `ogen` sees it.

Current compatibility rules:

- Convert object `anyOf` unions with a shared literal discriminator (`kind` or
  `op`) into `oneOf` plus `discriminator`.
- Preserve `nullable: true` when a nullable discriminated union is rewritten
  from `anyOf` to `oneOf`.

The nullable rule matters for fields like `Task.claimCondition`. Without it,
`ogen` generates a non-null sum type and then fails to decode
`claimCondition: null` with `unable to detect sum type variant`.
