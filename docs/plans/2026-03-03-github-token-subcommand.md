# Design: `github token` subcommand + agent rules

**Issue**: #346
**Date**: 2026-03-03

## Problem

The `moltnet github credential-helper` only works for git operations
(push/pull/fetch) via the git credential protocol. The `gh` CLI uses
`GH_TOKEN` and doesn't go through git credential helpers. Agents working
as GitHub Apps have no clean way to use `gh pr create`, `gh issue create`,
etc. with their app identity.

## Solution

Three deliverables:

### 1. `moltnet github token` (Go CLI)

New subcommand that prints a raw GitHub App installation access token to
stdout. Reuses existing `getInstallationToken()` from `github.go`.

```
moltnet github token --credentials .moltnet/legreffier/moltnet.json
# outputs: ghs_xxxxxxxxxxxx
```

Falls back to `MOLTNET_CREDENTIALS_PATH` env var when `--credentials` is
omitted.

### 2. `legreffier github token` (TS CLI)

Convenience subcommand that infers the credentials path from agent name.

```
legreffier github token --name legreffier
# resolves to: moltnet github token --credentials .moltnet/legreffier/moltnet.json
```

Agent name resolution order:

1. `--name` flag
2. `GIT_CONFIG_GLOBAL` pattern (`.moltnet/<name>/gitconfig`)

Shells out to `npx @themoltnet/cli github token`.

### 3. Agent rules generated by `legreffier setup`

Each adapter gets a new `writeRules()` method. The rule instructs any AI
agent to prefix `gh` CLI commands with `GH_TOKEN=$(...)` when
`GIT_CONFIG_GLOBAL` points to a moltnet gitconfig.

Rule locations per agent:

- **Claude**: `.claude/rules/legreffier-gh.md`
- **Codex**: `.codex/rules/legreffier-gh.md`

Rule content (portable across agents):

> When `GIT_CONFIG_GLOBAL` is set to a `.moltnet/*/gitconfig` path,
> prefix all `gh` CLI commands with
> `GH_TOKEN=$(moltnet github token --credentials <creds-path>)`
> where `<creds-path>` is `.moltnet/<agent-name>/moltnet.json` and
> `<agent-name>` is extracted from the gitconfig path.

This approach is portable across Claude, Codex, and Copilot ‚Äî all
support rules/instructions files.

## Files changed

### Go CLI (`cmd/moltnet/`)

- `github.go` ‚Äî add `runGitHubToken()` function
- `github_test.go` ‚Äî add tests for `runGitHubToken()`
- `main.go` ‚Äî wire `token` subcommand under `github`

### Legreffier CLI (`packages/legreffier-cli/`)

- `src/index.tsx` ‚Äî add `github token` subcommand dispatch
- `src/adapters/types.ts` ‚Äî add `writeRules()` to `AgentAdapter` interface
- `src/adapters/claude.ts` ‚Äî implement `writeRules()` for Claude
- `src/adapters/codex.ts` ‚Äî implement `writeRules()` for Codex
- `src/setup.ts` ‚Äî add `writeRule()` helper, add `Bash(moltnet github token *)` to permissions
- `src/SetupApp.tsx` ‚Äî call `adapter.writeRules()` in setup flow

## Non-goals

- No hook-based interception (not portable across agents)
- No `GH_TOKEN` caching/refresh (tokens are short-lived, ~1h)
- No Copilot adapter (can be added later following the same pattern)
