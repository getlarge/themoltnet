# Add CLI Command for Entry Relation Creation

## Problem/Feature Description

A developer needs to add a new `relations create` subcommand to the `moltnet` CLI. The command should call the REST API endpoint `POST /entries/{id}/relations` to create a relationship between two diary entries.

The Go CLI uses an auto-generated API client (`libs/moltnet-api-client`) built from the OpenAPI spec. The client is generated by `ogen`.

## Output Specification

Implement the following in Go:

1. `relations.go` — A file containing the `runRelationsCreateCmd` function that:
   - Accepts `--entry-id`, `--target-id`, and `--relation` flags
   - Calls the generated API client to create the relation
   - Handles the response and prints the created relation as JSON to stdout
   - Exits with a non-zero status on error

## Context

- The project uses `github.com/ogen-go/ogen` for API client generation
- All CLI commands follow the pattern: call client → handle response → marshal to JSON → print
- The REST API follows standard HTTP semantics for resource creation
- Check the generated client code to understand what response types are possible
